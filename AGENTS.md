# AGENTS.md

本文件是这个项目的协作规则，后续参与本项目的智能体或助手都应遵守。

## 基本要求

1. 每次回答都要尽量用用户能听懂的话解释，少用生硬术语。
2. 说明修改内容时，要直接说清楚“改了什么、为什么改、会有什么影响”。
3. 不要乱删文件、代码或配置。
4. 如果确实需要删除、移动、覆盖文件，必须先明确说明原因，再等用户确认。
5. 优先做最小必要修改，避免顺手改动无关内容。
6. 如果发现风险、不确定项或可能影响现有功能的地方，要先提醒用户。

## 文件操作规则

1. 只在当前任务明确需要的范围内创建或修改文件。
2. 不要批量重命名、批量删除、清空目录，除非用户明确要求。
3. 修改前要先理解相关文件用途，避免误伤现有逻辑。
4. 如果目录里已经有类似文件，优先保持兼容，不随意替换。

## 回答风格

1. 先说结论，再补充必要解释。
2. 解释尽量简短、直白、具体。
3. 如果有专业词，后面要顺手补一句通俗解释。
